# Atelier Digital — Design System

This Storybook documents the design system that powers [atelierdigital.co.uk](https://atelierdigital.co.uk).
It was designed in **Figma**, tokenised with **Style Dictionary**, and implemented in **React + Tailwind CSS**
— all in collaboration with **Claude Code**.

---

## Token pipeline

Tokens flow through three layers. Every value in the codebase traces back to a named token.

<div style={{ display: 'flex', flexDirection: 'column', gap: 8, margin: '16px 0 24px' }}>
  <Layer
    label="Primitives"
    example="tokens.json → base.css"
    description="Raw values — full colour palettes, numeric spacing steps, type sizes, radius and shadow definitions. No semantic meaning, just named values. Shared between Figma variables and the codebase."
  />
  <Layer
    label="Semantic tokens"
    example="semantic.css"
    description="Intent-based assignments — --fg-primary, --bg-surface, --accent-bg. Each token has a light and dark mode value. Components only ever reference semantic tokens, never primitives directly."
  />
  <Layer
    label="Tailwind utilities"
    example="tailwind.config.js (auto-generated)"
    description="Every semantic token becomes a Tailwind class. bg-bg-surface, text-fg-muted, rounded-radius-4, shadow-s. No arbitrary values in production markup."
  />
</div>

To regenerate after changing `tokens.json`:

<CodeBlock>npm run tokens</CodeBlock>

---

## Dark mode

The `.dark` class on `<html>` remaps every semantic token. No component knows what mode it's in.

<CodeBlock>{`/* semantic.css */
:root        { --bg-page: #ffffff; --fg-primary: #1f1f1f; }
.dark        { --bg-page: #141414; --fg-primary: #fafafa; }

/* Component — unchanged between modes */
<div className="bg-bg-page text-fg-primary" />`}</CodeBlock>

Use the **Color scheme** toggle in the Storybook toolbar (top right) to switch modes.

---

## Writing stories

Stories can use any Tailwind class or CSS custom property. The decorator wraps every story in
`bg-bg-page text-fg-primary`, so tokens resolve correctly out of the box.

<CodeBlock>{`// MyComponent.stories.jsx
export default {
  title: 'Components / MyComponent',
  component: MyComponent,
};

export const Default = {
  render: () => (
    <div className="p-6 rounded-radius-6 bg-bg-surface border border-border-subtle shadow-s">
      <p className="text-copy-m text-fg-secondary">Uses design system tokens</p>
    </div>
  ),
};`}</CodeBlock>

## Key token groups

| Group | Example classes | Purpose |
|---|---|---|
| Background | `bg-bg-page` `bg-bg-surface` | Page and card surfaces |
| Foreground | `text-fg-primary` `text-fg-muted` | Text hierarchy |
| Accent | `bg-accent-bg` `text-accent-fg` | Primary actions, links |
| Border | `border-border-subtle` `border-glass-subtle` | Dividers, outlines |
| Feedback | `bg-feedback-success-bg` | Status states |
| Palette | `bg-palette-purple-bg` | Tinted pill/badge colours |
| Spacing | `p-space-4` `gap-space-6` | Consistent spacing |
| Radius | `rounded-radius-4` `rounded-radius-8` | Border radii |
| Shadow | `shadow-xs` `shadow-s` `shadow-m` | Elevation |
| Typography | `text-h2` `text-copy-m` `text-label-s` | Type scale |
